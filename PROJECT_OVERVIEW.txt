  1. PROJECT SUMMARY
================================================================================

  This project is a fully custom-built, professional website designed for
  IMPACT GRAPHICS & SON SARL — a creative branding and visual communication
  agency based in Akwa, Douala, Cameroon.

  The website serves as the company's digital storefront. It showcases the
  agency's services (branding, printing, graphic design, photography, video
  production, web design, and more), displays a portfolio of past work,
  provides a way for potential clients to request quotes and book services,
  features a blog with educational articles about branding, and offers
  multiple ways for visitors to get in touch — including direct WhatsApp
  integration.

  The website was built using modern, industry-standard web technologies to
  ensure it loads quickly, looks stunning on any device (phone, tablet, or
  computer), and delivers a smooth, professional user experience that
  reflects the quality of IMPACT GRAPHICS' work.

  2. WHAT THE WEBSITE DOES (IN PLAIN LANGUAGE)
================================================================================

  Imagine walking into a beautifully decorated showroom where everything is
  laid out for you — the company's story, their services with pricing, photos
  of their past work, articles to help you understand branding, and a simple
  way to request a quote. That is exactly what this website does, but online.

  When someone visits the website, they are greeted with a visually striking
  homepage that immediately communicates what IMPACT GRAPHICS does. From
  there, they can:

    •  Learn about the company's history, mission, vision, and values
    •  Browse all 13 services with descriptions, pricing, and turnaround times
    •  View a filterable gallery of past creative projects
    •  Read educational blog articles about branding and design
    •  Book a service or request a quote (which sends directly to WhatsApp)
    •  Contact the company via a form, phone, email, or social media
    •  Click a floating WhatsApp button on any page for instant messaging

  Everything is designed to convert a casual visitor into a paying client.

  5. WHY THESE TECHNOLOGIES WERE CHOSEN
================================================================================

  Every technology was chosen for a specific reason. Here is the logic:

  REACT — was chosen because it is the world's most popular framework for
  building websites. It is maintained by Meta (Facebook) and used by
  companies like Netflix, Airbnb, and Instagram. React makes it easy to
  build complex, interactive websites that are fast and maintainable. Its
  "component-based" approach means parts of the website (like the navbar,
  footer, service cards) are built once and reused everywhere, saving
  development time and ensuring consistency.

  TYPESCRIPT — was chosen over plain JavaScript because it adds a safety
  net. When building a large website with many files, it is easy to make
  mistakes (like sending a number where text is expected). TypeScript
  catches these mistakes automatically before the website even runs,
  reducing bugs and making the code more reliable and easier to maintain
  in the future.

  VITE — was chosen over older tools (like Webpack) because it is
  dramatically faster. During development, changes appear on screen
  almost instantly (under 100 milliseconds). When building the final
  version for the internet, Vite produces highly optimized, compact
  files that load quickly for visitors.

  TAILWIND CSS — was chosen because it speeds up design work enormously.
  Instead of switching between code files and style files, the developer
  styles everything directly in the same file. This means design changes
  that would normally take hours take minutes. The final website also
  only includes the styles it actually uses, resulting in a very small
  file size (fast loading).

  SHADCN/UI — was chosen to provide professionally designed interface
  elements. Rather than designing every button, dialog, and form from
  scratch, shadcn/ui provides beautiful, accessible components that
  follow modern design standards. This saved significant development
  time while ensuring the website looks polished and premium.

  VERCEL — was chosen for hosting because it is purpose-built for React
  websites. It provides automatic HTTPS security, a global content
  delivery network (meaning the website loads fast no matter where in
  the world a visitor is), and zero-configuration deployment. It is
  also used by companies like The Washington Post, Uber, and Nike.

  8. TYPOGRAPHY (FONTS)
================================================================================

  The website uses two professional Google Fonts, each serving a specific
  purpose:

  POPPINS (Headings & Titles)
  ────────────────────────────
  •  Used for: All headings (h1, h2, h3, h4, h5, h6)
  •  Weights: 400 (Regular), 500 (Medium), 600 (Semi-Bold), 700 (Bold)
  •  Why: Poppins is a geometric sans-serif font with clean, rounded
     letterforms. It feels modern and friendly while remaining professional.
     Its bold weight commands attention in headings without feeling aggressive.
  •  Example: Section titles like "Our Services", "Our Portfolio", page
     titles like "About IMPACT GRAPHICS"

  INTER (Body Text & Navigation)
  ───────────────────────────────
  •  Used for: Body paragraphs, navigation links, button text, descriptions
  •  Weights: 300 (Light), 400 (Regular), 500 (Medium), 600 (Semi-Bold),
     700 (Bold)
  •  Why: Inter was specifically designed for computer screens. It is
     extremely legible at small sizes and has excellent spacing between
     letters, making long paragraphs comfortable to read. It is used by
     companies like GitHub and Figma.
  •  Example: Service descriptions, blog article text, form labels

  Both fonts are loaded from Google Fonts using "preconnect" — a technique
  that tells the browser to start loading the fonts early, so they appear
  instantly when the page opens (no flash of unstyled text).

  9. DESIGN STRUCTURE & LAYOUT
================================================================================

  The website follows a modern "single-page application" (SPA) design
  pattern. This means:

  •  There is one HTML file (index.html) that acts as the "shell"
  •  React dynamically swaps the content inside this shell when the
     visitor navigates between pages
  •  This makes page transitions feel instant — there is no white screen
     or loading between pages

  14. RESPONSIVE DESIGN (MOBILE, TABLET, DESKTOP)
================================================================================

  The website is fully responsive, meaning it automatically adjusts its
  layout to look perfect on any screen size:

  DESKTOP (1024px and above):
  •  Full navigation bar with all links visible
  •  Content arranged in multi-column grids (3-4 columns)
  •  Side-by-side text and image layouts
  •  Full-width hero sections with large typography

  •  Navigation may show hamburger menu
  •  Grids reduce to 2 columns
  •  Images and text may stack vertically

  MOBILE (below 768px):
  •  Hamburger menu replaces the navigation links
  •  All content stacks into a single column
  •  Typography sizes reduce for readability
  •  Buttons become full-width for easy tapping
  •  Touch-friendly spacing between interactive elements
  •  WhatsApp button positioned safely with "safe area" padding
    (accounting for phone notches and rounded corners)

  The responsive behavior is handled by Tailwind CSS breakpoint prefixes:
  •  "sm:" applies at 640px and above
  •  "md:" applies at 768px and above
  •  "lg:" applies at 1024px and above
  •  "xl:" applies at 1280px and above

  15. DEPLOYMENT & HOSTING
================================================================================

  The website is deployed (published) on Vercel, a professional cloud
  hosting platform. Here is how the deployment pipeline works:

  1. The developer writes and tests code locally on their computer
  2. The code is pushed to a Git repository (version control)
  3. Vercel automatically detects the new code and builds the website
  4. The built website is distributed to Vercel's global network of
     servers (data centers around the world)
  5. When a visitor anywhere in the world opens the website, they are
     served from the server closest to them — this means fast loading
     whether the visitor is in Douala, Paris, or New York

  The vercel.json configuration file contains a "rewrite" rule that
  ensures all URL paths (like /about, /services, /portfolio) are handled
  by the single index.html file. This is necessary because the website
  is a "single-page application" — all routing is handled by React in
  the browser, not by the server.

  Security: The website is served over HTTPS (the padlock icon in the
  browser), which encrypts all data between the visitor's browser and
  the server. This is provided automatically by Vercel at no extra cost.

  18. IMPORTANT NOTES
================================================================================

  1. NO BACKEND SERVER OR DATABASE
     This website is a "static" front-end application. It does not have
     a backend server or database. All form submissions go directly to
     WhatsApp. This means:
     •  No server hosting costs
     •  No database maintenance required
     •  No risk of server downtime affecting the website
     •  The website loads extremely fast
     If in the future a backend is needed (e.g., for a client portal,
     appointment booking system, or e-commerce), one can be added without
     rebuilding the front-end.

  2. ALL CONTENT IS HARD-CODED
     Blog articles, service listings, portfolio items, testimonials, and
     pricing are written directly in the code. To update them, a developer
     needs to edit the relevant file, rebuild, and redeploy. In the future,
     a content management system (CMS) like Strapi or Sanity could be
     integrated to allow non-technical staff to update content.

  3. PRICING IS IN XAF (CFA FRANC)
     All prices are displayed in the local currency (XAF / CFA Franc),
     appropriate for the Cameroon market.

  4. BILINGUAL CONSIDERATION
     The current website is in English. If a French version is needed
     (Cameroon is bilingual), internationalization (i18n) can be added
     to support multiple languages without rebuilding the site.

  5. PERFORMANCE
     The website is built for speed:
     •  Lazy loading of images (only loads when scrolled into view)
     •  Optimized build output (minified and compressed code)
     •  Global CDN delivery via Vercel
     •  Efficient CSS with Tailwind (only includes used styles)
     •  Modern React 19 with improved rendering performance

  6. BROWSER COMPATIBILITY
     The website works on all modern browsers:
     •  Google Chrome (desktop and mobile)
     •  Mozilla Firefox
     •  Apple Safari (desktop and iOS)
     •  Microsoft Edge
     •  Samsung Internet
     •  Opera
     Autoprefixer automatically adds vendor-specific CSS rules to ensure
     compatibility.

  7. ACCESSIBILITY
     The Radix UI components (used under the hood by shadcn/ui) follow
     WAI-ARIA accessibility standards, meaning the website is usable
     by people with disabilities using screen readers and keyboard
     navigation.

  8. VERSION CONTROL
     The project uses Git for version control (the .git folder). This
     means every change to the code is tracked, can be reviewed, and
     can be rolled back if needed. It is like having an "undo" history

  9. CODE QUALITY

================================================================================
  19. GLOSSARY OF TECHNICAL TERMS
================================================================================

  For reference, here are plain-language definitions of technical terms
  used throughout this document:

  API (Application Programming Interface)
    A set of rules that allows different software programs to communicate
    with each other. Like a waiter who takes your order (request) to the
    kitchen (server) and brings back your food (response).

  BACKEND
    The behind-the-scenes part of a website that users do not see. It
    handles data storage, business logic, and server communication. This
    project does not have a backend.

  BUILD
    The process of converting raw code into an optimized, compressed
    version ready for the internet. Like compiling a book manuscript
    into a printed, bound book.

  CDN (Content Delivery Network)
    A network of servers around the world that stores copies of the
    website. When someone visits the site, they get the copy from the
    server closest to them, making it load faster.

  CMS (Content Management System)
    A tool that allows non-technical people to update website content
    (text, images, blog posts) without writing code. Like WordPress,
    but more modern.

  COMPONENT
    A self-contained, reusable piece of the website. For example, the
    Navbar is a component — it is built once and appears on every page.
    Like a LEGO brick that can be placed anywhere.

  CSS (Cascading Style Sheets)
    The language that controls how a website looks — colors, fonts, sizes,
    spacing, and layouts. If HTML is the skeleton, CSS is the skin and
    clothing.

  DEPLOYMENT
    The process of putting the website on the internet so the public can
    access it. Like opening the doors to a new store.

  FRAMEWORK
    A pre-built foundation of tools and patterns that developers use to
    build applications faster. Like using a house blueprint instead of
    designing every wall from scratch.

  FRONTEND
    The part of the website that visitors see and interact with —
    everything you see in the browser. This entire project is a
    frontend application.

  GIT
    A version control system that tracks every change to the code.
    Like having a detailed logbook of every edit ever made, with the
    ability to go back to any previous version.

  HTML (HyperText Markup Language)
    The language that defines the structure and content of a web page.
    It is the skeleton of every website — headings, paragraphs, images,
    links, and forms are all defined in HTML.

  HOSTING
    The service that stores the website files on a server connected to
    the internet, making it accessible to visitors worldwide.

  HTTPS
    A secure version of HTTP (the protocol used to load websites).
    The "S" stands for "Secure." It encrypts data between the visitor
    and the server, shown by the padlock icon in the browser.

  LAZY LOADING
    A technique where images and content only load when the visitor
    scrolls to them, rather than loading everything at once. This makes
    the initial page load much faster.

  MODAL / DIALOG
    A window that appears on top of the page content, dimming the
    background. Used on the Blog page to display full articles without
    navigating away.

  RESPONSIVE DESIGN
    A design approach where the website automatically adjusts its layout
    to look good on any screen size — from a small phone to a large
    desktop monitor.

  ROUTING
    The system that determines which page content to display based on
    the URL. When you type "/about" in the address bar, the router
    shows the About page.

  SPA (Single-Page Application)
    A website that loads one HTML page and dynamically updates the
    content as the user navigates, instead of loading entirely new
    pages from the server. This makes navigation feel instant.

  UI (User Interface)
    Everything the visitor sees and interacts with on screen — buttons,
    menus, forms, images, text, and animations.

  URL (Uniform Resource Locator)
    The web address typed into the browser to visit a page. For example:
    https://impactgraphics.com/services

  UTILITY CLASS
    In Tailwind CSS, a small, single-purpose CSS class like "text-white"
    (makes text white) or "rounded-lg" (adds rounded corners). Hundreds
    of these classes are combined to style elements.
